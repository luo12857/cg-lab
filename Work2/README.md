## 一、程序概述
程序核心特性：
- 基于 GPU 并行计算加速（Taichi 自动适配 CUDA/CPU 架构）；
- 完整实现模型、视图、投影三大变换矩阵的推导与计算；
- 支持交互式旋转、渲染模式切换，可视化 3D 到 2D 屏幕的映射过程；
- 采用 Bresenham 直线算法实现像素级线段绘制，保证渲染精度。

## 二、核心功能模块
### 1. 基础初始化模块
| 功能点 | 说明 |
|--------|------|
| 计算架构初始化 | `ti.init(arch=ti.gpu)`：优先使用 GPU 加速渲染，大幅提升像素计算效率 |
| 画布定义 | `ti.Vector.field(3, ti.f32, RESOLUTION)`：创建 700×700 分辨率的 RGB 三通道像素场，用于存储彩色渲染结果 |
| 立方体数据初始化 | `init_cube_data()` 内核：预定义立方体的 8 个顶点（齐次坐标）和 12 条边的拓扑关系，存储在全局场中避免重复创建 |

### 2. MVP 变换矩阵模块（核心）
实现 3D 图形学的核心变换流程，所有矩阵均为 4×4 齐次矩阵（支持平移+旋转+投影）：

| 矩阵类型 | 函数名 | 核心作用 |
|----------|--------|----------|
| 模型矩阵 | `get_model_matrix(angle)` | 实现模型绕 Z 轴旋转，输入角度制数值，自动转换为弧度制计算 |
| 视图矩阵 | `get_view_matrix(eye_pos)` | 将相机（观察者）平移至世界坐标系原点，模拟相机视角（实验中相机固定在 (0,0,5)） |
| 投影矩阵 | `get_projection_matrix(eye_fov, aspect_ratio, zNear, zFar)` | 实现透视投影：<br>1. 透视平截头体 → 正交长方体；<br>2. 正交长方体 → NDC（标准设备坐标系，范围 [-1,1]） |

### 3. 辅助渲染模块
| 函数名 | 功能 | 关键细节 |
|--------|------|----------|
| `draw_line(p1, p2, color)` | 基于 Bresenham 算法绘制线段 | 1. 整数坐标计算，避免浮点误差；<br>2. 边界检测，仅绘制窗口内像素；<br>3. 支持自定义 RGB 颜色 |
| `ndc2screen(v)` | NDC 坐标转屏幕像素坐标 | 将 [-1,1] 范围的 NDC 坐标映射到 700×700 窗口的像素坐标 |

### 4. 核心渲染模块
`render(angle, render_cube)` 内核实现完整渲染流程，支持两种渲染模式：

| 渲染模式 | 触发条件 | 渲染逻辑 | 视觉效果 |
|----------|----------|----------|----------|
| 3D 三角形 | `render_cube=0` | 1. 定义三角形 3 个顶点；<br>2. 执行 MVP 变换 + 透视除法；<br>3. 转屏幕坐标后绘制三条边 | 三条边分别为红、绿、蓝三色 |
| 3D 立方体 | `render_cube=1` | 1. 遍历立方体 12 条边；<br>2. 对每条边的两个顶点执行 MVP 变换 + 透视除法；<br>3. 转屏幕坐标后绘制所有边 | 白色线框立方体，支持旋转 |

### 5. 交互控制模块
通过 Taichi GUI 实现键盘交互，实时控制渲染效果：

| 按键 | 功能 |
|------|------|
| `ESC` | 退出程序 |
| `c` | 切换渲染模式（三角形 ↔ 立方体） |
| `A` | 模型绕 Z 轴顺时针旋转（角度 +1°） |
| `D` | 模型绕 Z 轴逆时针旋转（角度 -1°） |

## 三、渲染流程（核心链路）
### 1. 视觉效果
- 三角形模式：显示彩色边的 3D 三角形，旋转时呈现透视效果；
- 立方体模式：显示白色线框立方体，旋转时可清晰观察 3D 透视变化；
- 所有渲染结果均为实时生成，旋转操作流畅无卡顿。
