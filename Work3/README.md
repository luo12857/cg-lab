## 一、实验目的

1. 理解贝塞尔曲线（Bézier Curve）的数学原理与 De Casteljau 递推算法。
2. 掌握使用 **Taichi** 框架进行 GPU 并行编程，实现高性能图形渲染。
3. 实现一个可交互的贝塞尔曲线绘制程序，支持鼠标添加控制点、实时绘制曲线。

## 二、实验原理

### 2.1 贝塞尔曲线

贝塞尔曲线是由法国工程师 Pierre Bézier 在 1962 年提出的一种参数曲线，广泛应用于计算机图形学中。其定义为：

给定 $n+1$ 个控制点 $P_0, P_1, \dots, P_n$，$n$ 次贝塞尔曲线的参数方程为：

$$
B(t) = \sum_{i=0}^{n} \binom{n}{i} (1-t)^{n-i} t^i P_i, \quad t \in [0, 1]
$$

### 2.2 De Casteljau 算法

De Casteljau 算法是计算贝塞尔曲线上点的递归方法，直观且数值稳定：

> 对参数 $t$，在每条相邻控制点连线上按比例 $t : (1-t)$ 取分点，得到 $n$ 个新点；重复此过程直到只剩一个点，即为曲线上对应 $t$ 的点。

```
P₀ ─── P₁ ─── P₂ ─── P₃      (原始控制点)
 ↓      ↓      ↓
Q₀ ─── Q₁ ─── Q₂              (第一轮插值)
 ↓      ↓
R₀ ─── R₁                     (第二轮插值)
 ↓
S₀                            (最终曲线点)
```

### 2.3 Taichi GPU 加速

Taichi 是一个嵌入 Python 的并行编程语言，可将 `@ti.kernel` 装饰的函数自动编译到 GPU 上执行。本实验利用 Taichi 实现：

- **像素清空** — GPU 并行将 $800 \times 800$ 像素全部置零。
- **曲线绘制** — 将 1001 个曲线采样点通过 GPU 并行写入像素缓冲区。
- **性能优化** — 先在 CPU 端批量计算所有曲线点坐标，通过 `from_numpy()` 一次性打包传入 GPU，避免逐点通信开销。

## 四、实验内容与实现

### 4.1 功能概述

- 🖱️ **鼠标左键点击**：在点击位置添加控制点（最多 100 个）。
- 🔄 **实时绘制**：每次添加控制点后自动重绘贝塞尔曲线。
- 🎯 **控制点与多边形显示**：红色圆点标记控制点，灰色连线显示控制多边形。
- ⌨️ **按 `C` 键清空**：重置所有控制点。
- 🚀 **GPU 加速**：利用 Taichi kernel 在 GPU 上并行绘制，保持 60 FPS 流畅交互。

### 4.2 代码结构

```
贝塞尔曲线.py
├── 初始化：ti.init(arch=ti.gpu)        # 启用 GPU 后端
├── 全局字段
│   ├── pixels          # 像素缓冲区 (800×800)
│   ├── gui_points      # GUI 控制点缓冲区
│   ├── gui_indices     # 线段索引缓冲区
│   └── curve_points_field  # 曲线坐标 GPU 缓冲区（性能优化关键）
├── de_casteljau()      # 纯 Python 递归实现 De Casteljau 算法
├── clear_pixels()      # @ti.kernel GPU 并行清屏
├── draw_curve_kernel() # @ti.kernel GPU 并行绘制曲线点
└── main()              # 主循环：事件处理 + 坐标计算 + GPU 绘制
```

### 4.3 性能优化要点

1. **减少 CPU↔GPU 通信次数**：将所有曲线点先算好存入 NumPy 数组，通过 `from_numpy()` 一次性传输，而非逐点传输。
2. **GPU 并行绘制**：`draw_curve_kernel` 中的 `for i in range(n)` 由 Taichi 自动在 GPU 上并行执行，1001 个点的绘制近乎瞬时完成。
3. **GPU 并行清屏**：`clear_pixels` 在 GPU 上并行清空所有像素，避免 CPU 循环开销。

## 六、总结与体会

1. **De Casteljau 算法思想**：通过反复线性插值递推得到曲线上任意点，算法简洁且编程实现直观，适用于任意次数的贝塞尔曲线。
2. **GPU 编程实践**：通过 Taichi 框架亲身体验了 GPU 并行计算带来的性能提升——将原本 CPU 逐像素绘制的瓶颈转化为 GPU 极速并行绘制，使交互帧率稳定在 60 FPS。
3. **数据通信优化**：理解了 CPU 与 GPU 之间的数据传输是性能关键瓶颈，批量传输（batch transfer）相比逐点传输能带来数量级的性能改善。
4. **问题解决**：解决了因频繁 `from_numpy` 造成的帧率骤降问题，将曲线点计算与 GPU 数据传输分离，保证了交互流畅性。
