## 一、实验目的

1. **理论理解**：理解并掌握局部光照的基本原理，区分环境光（Ambient）、漫反射（Diffuse）和镜面高光（Specular）三种光照分量的物理含义与数学表达。
2. **数学基础**：熟练掌握三维空间中的向量运算，包括法向量计算、光线方向、视线方向与反射向量的推导与应用。
3. **工程实践**：掌握如何利用 Taichi 实现交互式渲染，通过 UI 控件实时调节材质参数，直观感受各个参数对渲染结果的影响。

## 二、实验原理

### 2.1 局部光照模型

**Phong 光照模型**属于**局部光照模型**，仅考虑光源到物体的直接光照，忽略物体间的光线弹射。它将物体表面某点的最终颜色分解为三个独立分量的线性叠加：

```
I = I_ambient + I_diffuse + I_specular
```

#### 2.1.1 环境光 (Ambient)

**物理含义**：模拟场景中光线经环境多次散射后来自各个方向的间接光照的近似，是一个常数项。

```
I_ambient = Ka × I_light × C_obj
```

| 符号 | 含义 | 取值范围 |
|------|------|----------|
| `Ka` | 环境光反射系数（材质属性） | [0, 1] |
| `I_light` | 光源颜色 | RGB [0, 1] |
| `C_obj` | 物体表面颜色 | RGB [0, 1] |

- 不依赖于光源位置和视线方向
- 确保物体背光面也能被看见，避免出现纯黑区域

#### 2.1.2 漫反射 (Diffuse)

**物理含义**：粗糙表面向各个方向均匀反射光线的现象，遵循 **Lambert 余弦定律**——表面接收到的光照强度与光线入射角 θ 的余弦成正比。

```
I_diffuse = Kd × max(0, N·L) × I_light × C_obj
```

| 符号 | 含义 |
|------|------|
| `N` | 表面法向量（单位向量） |
| `L` | 从交点指向光源的方向向量（单位向量） |
| `N·L` | 两向量点积，等于 cos θ |
| `max(0, ...)` | 光线从背面入射时（cos θ < 0），漫反射贡献为 0 |

- 依赖光源位置和法线方向
- **不依赖**观察者（相机）位置——从任何角度看漫反射效果相同

#### 2.1.3 镜面高光 (Specular)

**物理含义**：光滑表面将光线集中反射到特定方向，形成明亮的高光斑。视线方向与反射光方向越接近，看到的高光越强。

```
I_specular = Ks × max(0, R·V)ⁿ × I_light
```

| 符号 | 含义 |
|------|------|
| `R` | 光线经表面反射后的方向向量 |
| `V` | 从交点指向观察者（相机）的方向向量 |
| `R·V` | 反射方向与视线方向的接近程度 |
| `n` (Shininess) | 高光指数，控制高光斑的集中程度 |

- 同时依赖光源位置、法线方向和观察方向
- `n` 越大，高光斑越小越亮（金属感强）；`n` 越小，高光斑越大越柔和（塑料感强）

#### 2.1.4 三种光照对比

| 分量 | 依赖光源位置？ | 依赖视线方向？ | 依赖法线方向？ | 视觉效果 |
|------|:---:|:---:|:---:|----------|
| **Ambient** | ❌ | ❌ | ❌ | 均匀照亮，无方向感 |
| **Diffuse** | ✅ | ❌ | ✅ | 明暗过渡，立体感 |
| **Specular** | ✅ | ✅ | ✅ | 高光斑，光泽感 |

### 2.2 三维空间向量运算

#### 2.2.1 核心向量

```
     lightPos (光源)
        │
        │ L
        │
        ↓              V (视线方向)
    ┌───────┐          ↑
    │   P   │──────────┘
    │   ↓N  │        camera (相机)
    │   │   │
    └───────┘
        ↓ R (反射方向)
```

| 向量 | 符号 | 定义 | 计算公式 |
|------|------|------|----------|
| 法向量 | `N` | 表面在该点的垂直方向 | `normalize(p - center)`（球体） |
| 光源方向 | `L` | 从交点指向光源 | `normalize(lightPos - p)` |
| 视线方向 | `V` | 从交点指向相机 | `normalize(ro - p)` |
| 反射方向 | `R` | 光线在表面上的镜面反射方向 | `reflect(-L, N)` |

#### 2.2.2 向量点积

```python
a · b = |a||b|cosθ = a.x·b.x + a.y·b.y + a.z·b.z
```

在 Phong 模型中的应用：
- **N·L** = cos θ —— 判断表面朝向光源的程度（决定漫反射强度）
- **R·V** = cos φ —— 判断反射方向与视线的一致程度（决定高光强度）

> 所有参与光照计算的方向向量（N, L, V, R）必须是**单位向量**，否则点积结果不能正确反映角度关系。

#### 2.2.3 向量归一化

```python
@ti.func
def normalize(v):
    return v / v.norm(1e-5)   # 除以模长，避免除零
```

#### 2.2.4 反射向量计算

给定入射方向 `I`（指向交点）和法线 `N`，反射方向 `R` 的公式为：

```
R = I - 2(I·N)N
```

代码实现：
```python
@ti.func
def reflect(I, N):
    return I - 2.0 * I.dot(N) * N
```

> 代码中传入 `-L`（从光源指向交点）而非 `L`，因为反射公式的入射方向定义为**指向表面**的方向。

#### 2.2.5 法向量计算

**球体法线**：从球心指向交点
```python
normal = normalize(p - center)
```

**圆锥法线**：结合圆锥斜率和位置
```python
p_local = ro_local + rd * t
k = (radius / height) ** 2
normal = normalize(Vector([p_local.x, -k * p_local.y, p_local.z]))
```

### 2.3 光线与几何体求交

#### 球体相交

光线 `P(t) = ro + t·rd` 代入球体方程 `|P - center|² = radius²`：

```
A = 1           (rd 为单位向量)
B = 2·oc·rd    (oc = ro - center)
C = |oc|² - r²
Δ = B² - 4AC
```

判别式 `Δ > 0` 时有交点，取最小正数 t。

#### 圆锥相交

圆锥方程 `x² + z² = k·y²`，代入光线参数方程后同样化为二次方程求解，还需额外验证交点 y 坐标在 `[-H, 0]` 高度范围内。

## 三、实验内容

### 3.1 场景设置

| 物体 | 位置 | 颜色 | 几何参数 |
|------|------|------|----------|
| 红色球体 | 左侧 `(-1.2, -0.2, 0.0)` | `(0.8, 0.1, 0.1)` | 半径 1.2 |
| 紫色圆锥 | 右侧 `(1.2, 1.2, 0.0)` → `y=-1.4` | `(0.6, 0.2, 0.8)` | 底面半径 1.2，高度 2.6 |

- 光源位置：`(2.0, 3.0, 4.0)`（白色光源）
- 相机位置：`(0.0, 0.0, 5.0)`

### 3.2 渲染流水线

```
逐像素并行 (i, j)
    │
    ├─ (1) 生成光线：根据像素坐标计算视角光线方向 rd
    │
    ├─ (2) 求交测试：遍历场景中所有物体，找到最近交点
    │      ├─ intersect_sphere()
    │      └─ intersect_cone()
    │
    ├─ (3) 计算交点法向量 N
    │
    ├─ (4) Phong 光照着色
    │      ├─ ambient  = Ka × lightColor × objColor
    │      ├─ diffuse  = Kd × max(0, N·L) × lightColor × objColor
    │      ├─ specular = Ks × max(0, R·V)ⁿ × lightColor
    │      └─ color    = ambient + diffuse + specular
    │
    └─ (5) 写入 pixels[i, j]
```

### 3.3 Taichi 工程实现

| Taichi 特性 | 在本实验中的应用 |
|-------------|-----------------|
| `@ti.kernel` | `render()` 函数在 GPU 上并行计算每个像素的颜色 |
| `@ti.func` | 自定义向量运算和相交测试函数 |
| `ti.Vector.field` | 存储像素颜色值（RGB 浮点向量） |
| `ti.field` | 存储材质参数 Ka、Kd、Ks、Shininess，供 GPU 访问 |
| `ti.ui.Window` | 创建 GUI 窗口和画布 |
| `gui.slider_float()` | 实时交互滑块控件 |

### 3.4 实时交互控制

程序窗口右侧提供实时滑动条面板，拖动滑块即时更新参数，下一帧渲染立刻生效：

```python
with gui.sub_window("Material Parameters", 0.7, 0.05, 0.28, 0.22):
    Ka[None] = gui.slider_float('Ka (Ambient)', Ka[None], 0.0, 1.0)
    Kd[None] = gui.slider_float('Kd (Diffuse)', Kd[None], 0.0, 1.0)
    Ks[None] = gui.slider_float('Ks (Specular)', Ks[None], 0.0, 1.0)
    shininess[None] = gui.slider_float('N (Shininess)', shininess[None], 1.0, 128.0)
```

### 3.5 交互参数一览

| 参数 | 范围 | 默认值 | 调节效果 |
|------|------|--------|----------|
| **Ka** (环境光系数) | 0.0 ~ 1.0 | 0.2 | 控制环境光亮度，影响物体暗面 |
| **Kd** (漫反射系数) | 0.0 ~ 1.0 | 0.7 | 控制漫反射强度，影响立体感 |
| **Ks** (镜面反射系数) | 0.0 ~ 1.0 | 0.5 | 控制镜面高光强度 |
| **N** (高光指数) | 1.0 ~ 128.0 | 32.0 | 控制高光集中程度（越大越亮越集中） |

## 四、实验步骤

### 步骤 1：环境搭建

```bash
pip install taichi
python "Phong光照模型.py"
```

### 步骤 2：单独观察各光照分量

| 操作 | 参数设置 | 预期观察 |
|------|----------|----------|
| 仅环境光 | `Ka=0.5, Kd=0, Ks=0` | 物体呈现均匀颜色，无明暗变化和立体感 |
| 仅漫反射 | `Ka=0, Kd=0.8, Ks=0` | 物体呈现明显明暗过渡，立体感强，但无高光 |
| 仅镜面高光 | `Ka=0, Kd=0, Ks=1, N=64` | 仅光源反射方向可见高光斑，其余为黑色 |
| 完整 Phong | `Ka=0.2, Kd=0.7, Ks=0.5, N=32` | 三者叠加，立体且有光泽感 |

### 步骤 3：参数扫描实验

固定其他参数为默认值，逐个调节单个参数，观察变化：

| 实验 | 调节参数 | 固定参数 | 观察重点 |
|------|----------|----------|----------|
| 实验 A | Ka: 0 → 1 | Kd=0.7, Ks=0.5, N=32 | 物体暗面逐渐变亮，整体对比度下降 |
| 实验 B | Kd: 0 → 1 | Ka=0.2, Ks=0.5, N=32 | 明暗过渡逐渐增强，立体感提升 |
| 实验 C | Ks: 0 → 1 | Ka=0.2, Kd=0.7, N=32 | 高光斑从无到有，逐渐变亮 |
| 实验 D | N: 1 → 128 | Ka=0.2, Kd=0.7, Ks=0.5 | 高光斑从大而散变为小而亮 |

### 步骤 4：模拟真实材质

| 材质 | Ka | Kd | Ks | N | 效果特征 |
|------|:--:|:--:|:--:|:--:|----------|
| 哑光墙面 | 0.3 | 0.8 | 0.1 | 8 | 漫反射为主，几乎无高光 |
| 塑料球 | 0.2 | 0.6 | 0.4 | 32 | 有较柔和的高光斑 |
| 金属球 | 0.1 | 0.3 | 0.8 | 128 | 小而强烈的高光斑 |
